## 구현 체크리스트

### 1. 게임 로직 (index.html `ShellGame` 또는 관련 객체)
- 새 요소의 상태(state)가 필요한지: `moving` / `opened` / `broken` 외에 추가 상태 있는지
- 스폰 로직 변경이 필요한지: 기존 2비트 주기 스폰에 통합할지, 별도 스케줄 이벤트를 만들지
- 히트 판정 변경이 필요한지: `findHittable()` / `findNearest()` 수정 범위

### 2. 스프라이트 / CSS
- 새 시각 표현을 위한 스프라이트 프레임이 있는지, 없으면 CSS만으로 표현 가능한지
- 새 CSS 클래스가 필요한지: `.shell` 확장 or 새 클래스
- 애니메이션(`@keyframes`)이 추가로 필요한지

### 3. 효과음 (SFX)
- 기존 `hit_1~4.mp3` / `miss_1~2.mp3` 중 재활용 가능한 것이 있는지
- 새 사운드 파일이 필요하면 `SFX` 객체에 키 추가

### 4. 레벨 시스템 연동
- 레벨에 따라 새 요소의 빈도/속도/난이도가 달라져야 하는지
- `LevelSystem.levelUp()`에서 해당 파라미터를 조정해야 하는지

### 5. UI
- 플레이어에게 새 요소를 알려주는 시각적 힌트(튜토리얼, 아이콘 등)가 필요한지
